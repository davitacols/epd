What is EPD?

EPD solves the most frustrating problem in JavaScript dependency management: peer dependency conflicts. When traditional package managers fail with cryptic errors, EPD steps in with intelligent resolution algorithms that understand your project's needs.

Smart Resolution
Automatically resolves version conflicts using advanced algorithms that understand semantic versioning and compatibility matrices.
Multi-PM Support
Works seamlessly with npm, yarn, and pnpm. Automatically detects your preferred package manager.
Workspace Aware
Built for monorepos. Understands workspace structures and optimizes dependencies across packages.
Zero Config
Works out of the box. No configuration files, no setup required. Just install and run.
Health Monitoring
Continuous dependency health scoring based on maintenance activity, security status, and community adoption.
Bundle Analysis
Real-time bundle size impact analysis helps you make informed decisions about dependencies.

Why Choose EPD?

Traditional package managers treat peer dependencies as an afterthought, leading to installation failures and version conflicts. EPD was built specifically to solve this problem.

The Problem
When you run npm install and encounter peer dependency warnings or errors, you're left to manually resolve conflicts. This process is time-consuming, error-prone, and often results in incompatible versions.
The Solution
EPD analyzes your entire dependency tree, understands the relationships between packages, and automatically selects versions that satisfy the maximum number of requirements while maintaining compatibility.

Installation

Get EPD running in your project

Global Installation

Install EPD globally to use it across all your projects:

npm install -g epd
yarn global add epd
pnpm add -g epd

Project Installation

For CI/CD environments or project-specific usage:

# Add to devDependencies
npm install --save-dev epd

# Run via npx
npx epd install

# Add to package.json scripts
{
"scripts": {
"install:epd": "epd install",
"health": "epd health --score"
}
}

Basic Usage

The install command is EPD's core functionality. It analyzes your project, resolves conflicts, and installs dependencies with enhanced peer dependency handling.

# Install all dependencies from package.json
epd install

# Add specific packages
epd add react react-dom
epd add @types/node --dev

# Force specific package manager
epd install --pm=yarn

# Enable debug output
epd install --debug

Parameters

Parameter Type Description
packages string[] Optional list of packages to install
--pm npm|yarn|pnpm Force specific package manager
--debug boolean Enable verbose logging
--dev boolean Install as development dependency

How It Works

EPD's installation process is designed to be both intelligent and safe:

1. Analysis
Scans project structure, detects workspaces, and maps existing dependencies.
2. Collection
Gathers peer dependencies from all packages and identifies potential conflicts.
3. Resolution
Uses smart algorithms to find versions that satisfy maximum requirements.
4. Installation
Creates optimized package.json and runs package manager with resolved dependencies.
Safety First
EPD always creates a backup of your package.json before making changes. If anything goes wrong, your original configuration is automatically restored.
